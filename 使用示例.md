# 财联社电报工具使用示例

## 基本使用

### 1. 常规运行（推荐）
```bash
python cls_to_feishu.py
```
这个命令会：
- 获取当天的财联社电报
- 保存到每日文件（如：cls_2025-01-15.md）
- 自动生成最近5天的整合文件
- 发送飞书通知（如果配置了）
- 清理旧文件

### 2. 仅生成5天整合文件
```bash
python cls_to_feishu.py --summary
```
这个命令只会：
- 读取已有的每日文件
- 生成最近5天的整合文件到 `5days` 文件夹
- 不获取新数据，不发送通知

## 文件结构说明

运行后，你的文件夹结构会是这样：

```
财联社自动/
├── output/
│   ├── cls/                    # 每日文件目录
│   │   ├── cls_2025-01-15.md   # 今天的电报
│   │   ├── cls_2025-01-14.md   # 昨天的电报
│   │   └── ...                 # 其他日期的电报
│   └── 5days/                  # 5天整合文件目录
│       └── 财联社电报_最近5天_20250115_143022.md  # 只保留最新的1个整合文件
├── cls_to_feishu.py
└── README.md
```

## 使用建议

1. **定时任务**：建议设置定时任务每小时运行一次常规命令
2. **手动整合**：当你需要查看最近几天的汇总时，可以单独运行 `--summary` 命令
3. **文件管理**：程序会自动清理旧文件，无需手动管理
4. **数据备份**：重要数据建议定期备份到其他位置

## 常见问题

**Q: 5天整合文件会覆盖每日文件吗？**
A: 不会。5天整合文件保存在独立的 `5days` 文件夹中，与每日文件完全分离。

**Q: 如果某天没有数据怎么办？**
A: 整合文件会显示 "该日期文件不存在" 或 "该日期暂无电报数据"。

**Q: 可以修改保留的整合文件数量吗？**
A: 可以，在代码中修改 `_cleanup_old_summary_files` 方法的 `keep_count` 参数（默认为1）。
